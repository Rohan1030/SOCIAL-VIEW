## 🏗️ System Design — Notification Pipeline

> A quick look at the architecture behind my most complex project:

```
Client Request
      ↓
Express API  →  JWT Auth  →  Rate Limiter (Redis)  →  Idempotency Check
      ↓
BullMQ Queue  (HIGH / NORMAL / LOW priority)
      ↓
Worker Pool  (concurrency: 5)
      ↓
┌─────────────┬──────────────┬──────────────┐
│ 📧 Email    │ 📱 SMS       │ 🔔 Push      │
│ Gmail SMTP  │ Twilio       │ Firebase FCM │
└─────────────┴──────────────┴──────────────┘
      ↓                            ↓
MongoDB Logs               Socket.io (Real-time)
(status tracking)          (Redis adapter)
      ↓                            ↓
Retry × 3 → DLQ          React Dashboard
```

**Key concepts:** Decoupled async architecture · Exponential backoff · Dead Letter Queue · Distributed rate limiting · Exactly-once delivery · WebSocket horizontal scaling
